## 1. Project Overview

### 1.1 Project Concept
**AutoRevise** is a comprehensive web-based spaced repetition learning system designed to optimize student learning through:
- Smart flashcard management with AI-powered spaced repetition
- Category-based MCQ practice system
- Gamification with achievements and point tracking
- Progress analytics and performance insights

### 1.2 Problem Statement
Traditional studying methods often lead to:
- Inefficient review scheduling
- Difficulty tracking progress across subjects
- Lack of engagement and motivation
- No personalized learning experience

### 1.3 Solution Approach
AutoRevise solves these problems by:
- Implementing **SM-2 spaced repetition algorithm** for optimal review timing
- Providing **gamified learning** with points, streaks, and achievements
- Offering **detailed analytics** for performance tracking
- Supporting **bulk content upload** for efficient content management

## 6. Algorithm Implementation

### 6.1 SM-2 Spaced Repetition Algorithm

#### **Algorithm Purpose:**
The SM-2 (SuperMemo 2) algorithm optimizes learning by scheduling card reviews at increasing intervals based on user performance.

#### **Implementation:**

```python
def calculate_sm2(rating, current_interval, current_ease):
    """
    SM-2 Spaced Repetition Algorithm
    
    Parameters:
    - rating: User's self-assessment ('forgot', 'hard', 'good', 'easy')
    - current_interval: Days since last review
    - current_ease: Current ease factor (difficulty multiplier)
    
    Returns:
    - new_interval: Days until next review
    - new_ease: Updated ease factor
    """
    
    # Step 1: Convert rating to quality score (0-5 scale)
    quality_map = {
        'forgot': 0,  # Complete blackout
        'hard': 3,    # Correct with serious difficulty
        'good': 4,    # Correct with hesitation
        'easy': 5     # Perfect response
    }
    quality = quality_map[rating]
    
    # Step 2: Calculate new ease factor
    # Formula: EF' = EF + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))
    new_ease = current_ease + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
    new_ease = max(1.3, new_ease)  # Minimum ease factor is 1.3
    
    # Step 3: Calculate new interval
    if quality < 3:
        # Failed - restart learning
        new_interval = 1
    else:
        if current_interval == 0:
            new_interval = 1
        elif current_interval == 1:
            new_interval = 6
        else:
            # I(n) = I(n-1) * EF
            new_interval = round(current_interval * new_ease)
    
    return new_interval, round(new_ease, 2)
```

### 6.2 Password Security Implementation

#### **bcrypt Hashing Process:**

```python
import bcrypt

def hash_password(plain_password):
    """
    Secure password hashing using bcrypt
    
    Process:
    1. Generate random salt (cost factor 12)
    2. Hash password with salt using Blowfish
    3. Return combined salt+hash string
    """
    # Generate salt with work factor 12 (2^12 = 4096 iterations)
    salt = bcrypt.gensalt(rounds=12)
    
    # Hash password (encoded as bytes)
    password_hash = bcrypt.hashpw(
        plain_password.encode('utf-8'),
        salt
    )
    
    # Return as string for database storage
    return password_hash.decode('utf-8')

def verify_password(plain_password, stored_hash):
    """
    Verify password against stored hash
    
    Returns: True if match, False otherwise
    """
    return bcrypt.checkpw(
        plain_password.encode('utf-8'),
        stored_hash.encode('utf-8')
    )
```

#### **Security Features:**

1. **Salt**: Random value prevents rainbow table attacks
2. **Adaptive Function**: Configurable work factor for future-proofing
3. **Timing Attack Prevention**: Constant-time comparison
4. **Key Stretching**: Multiple iterations make brute-force expensive

### 8.1 Normalization Process

#### **First Normal Form (1NF):**
- âœ… All columns contain atomic values
- âœ… No repeating groups
- âœ… Each row is unique (primary key)

#### **Second Normal Form (2NF):**
- âœ… Satisfies 1NF
- âœ… No partial dependencies
- âœ… All non-key attributes depend on entire primary key

- âœ… No transitive dependencies
- âœ… Non-key attributes depend only on primary key

## 9. Frontend-Backend Integration

### 9.1 API Communication Pattern

#### **Frontend API Service Class:**

```javascript
class AutoReviseAPI {
    constructor() {
        this.baseURL = 'http://127.0.0.1:5000';
        this.user = this.loadUserFromStorage();
    }
    
    /**
     * Generic request handler
     * Handles: Authentication, Error handling, JSON parsing
     */
    async request(endpoint, options = {}) {
        const config = {
            method: options.method || 'GET',
            credentials: 'include',  // Send cookies
            headers: {
                'Content-Type': 'application/json',
            },
            ...options
        };
        
        try {
            const response = await fetch(this.baseURL + endpoint, config);
            
            // Handle HTTP errors
            if (!response.ok) {
                const error = await response.json();
                throw new Error(error.error || 'Request failed');
            }
            
            return await response.json();
        } catch (error) {
            console.error('API Error:', error);
            throw error;
        }
    }
    
    // Example: Login method
    async login(email, password) {
        const response = await this.request('/login', {
            method: 'POST',
            body: JSON.stringify({ email, password })
        });
        
        this.saveUserToStorage(response.user);
        return response;
    }
    
    // Example: Get decks
    async getDecks() {
        return await this.request('/decks');
    }
}

// Global API instance
const api = new AutoReviseAPI();
```

### 10.3 Performance Optimization

#### **Database Query Optimization:**

```sql
-- Before: N+1 Query Problem
SELECT * FROM Decks WHERE user_id = 1;
-- Then for each deck:
SELECT COUNT(*) FROM Cards WHERE deck_id = ?;

-- After: Single JOIN Query
SELECT 
    d.deck_id,
    d.deck_name,
    COUNT(c.card_id) as card_count
FROM Decks d
LEFT JOIN Cards c ON d.deck_id = c.deck_id
WHERE d.user_id = 1
GROUP BY d.deck_id;
```

// Debounce search input
const debounce = (func, delay) => {
    let timeoutId;
    return (...args) => {
        clearTimeout(timeoutId);
        timeoutId = setTimeout(() => func(...args), delay);
    };
};
